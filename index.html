Welcome to BioJSON community site.

BioJSON was introduced in Jalview 2.9 to serve as a standard JavaScript Object Notation (JSON) model for representing/ exchanging Bioinformatics Alignment data. This is the first version of the specification and we welcome all feedbacks to make BioJSON more standardised, sustainable and widely adopted worldwide. The main content section was created with Docson - a resource which uses a JSON schema to generate a beautified and interactive documentation.

Advantages of JSON/BioJSON

  • BioJSON can store very sophisticated sequence alignments along with their linked-data more than any other Bioinformatics data format
  • BioJSON can be easily embedded in HTML pages and some image formats
  • BioJSON is based on JSON which is very popular and can be parsed in almost all programming languages
  • Integrates seamlessly with the HTTP rest architecture
  • Very light weight, self describing and easy to understand
  • JSON format is text only and uses JavaScript syntax
  • JSON is hierarchical (i.e. it can contain values within values)
  • Linked data can be efficiently represented

More information about JSON can be found at http://json.org/. BioJSON currently has the ability to represent any Sequence Alignment data in Jalview along with their Annotations, Features, Groups, Hidden Sequences, Hidden columns, and custom application settings in a structured manner.

The BioJSON Schema

  • When the mouse is hovered over the elements in the BioJSON schema section, traversable elements become highlighted and clickable.
  • Click the “object” button or the plus “+” link to expand an object
  • Click the minus “-” to collapse an object
  • Click the parenthesis “{}” link to view the JSON schema which provides a more technical description of the data
  • When you expand an object the following information becomes visible: The variables in the object, their data types and the description of each variable. This exactly corresponds to how it appears in BioJSON

BioJSON Vision

Our vision is to arrive at a point where any Alignment data represented in JSON conforms to the BioJSON specification.
